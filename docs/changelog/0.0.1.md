# 0.0.1

**April 3, 2026**

First release. The foundation is live — search, monitor, draft, post.

## What's New

**Watchlist Monitor**
Add usernames to `watchlist.yaml`, run `bun run watch`. The agent polls every 45 seconds, alerts you the moment someone posts, drafts 3 algorithm-optimized replies, and lets you pick one. Copies to clipboard, opens in Chrome — Cmd+V and Post.

**Phoenix-Scored Reply Drafting**
Every reply is drafted against X's open-source recommendation algorithm. The engine scores across favorite, dwell, profile click, follow, and reply signals — and avoids negative signals (not-interested, block, mute, report) that tank visibility.

**Chrome Posting**
The X API blocks replies to posts with restricted conversation controls (403 Forbidden). Chrome doesn't. The agent reads via API, posts via browser — no restrictions on who you can reply to.

**Account Discovery**
Searches your niche keywords every 15 minutes to find new accounts worth engaging with. Surfaces posts with high engagement from accounts you don't follow yet.

**Conversation Tracking**
Monitors mentions of your account every 30 minutes so you never miss a reply or tag.

**x-engage Skill**
Installed to `~/.claude/skills/x-engage/`. Say "x engage", "find posts", or paste any x.com URL in Claude Code to trigger it.

**MCP Server**
8 tools for Claude Code integration: search_posts, get_post, get_thread, get_user_posts, post_reply, like_post, check_targets, get_engagement_stats.

## Safety

- 20 replies/day cap
- Never same author twice per day
- User approval before every post
- No links or hashtags in replies
- Working hours only (7am–11pm)
- SQLite tracking for all activity

## Stack

- TypeScript on Bun
- X API v2 (OAuth 1.0a + OAuth 2.0 PKCE)
- MCP server for Claude Code
- claude-in-chrome for browser posting
- SQLite for state tracking
- Shell scripts for daemon orchestration
